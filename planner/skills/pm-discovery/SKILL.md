# PM Discovery Skill

프로덕트 디스커버리 단계에서 문제 정의, 가설 검증, 기회 식별.

## When to Activate

- 새로운 기능/제품 아이디어 탐색
- 사용자 문제 정의
- 가설 수립 및 검증 계획
- 기회 평가

## Discovery Framework

### 1. 문제 정의
```markdown
### Problem Statement
**누가**: [사용자 세그먼트]
**무엇을**: [하려고 할 때]
**문제**: [어떤 어려움을 겪는다]
**왜 중요한가**: [비즈니스 임팩트]
**근거**: [데이터/피드백]
```

### 2. 가설 수립
```markdown
### Hypothesis
우리는 [해결책]을 제공하면,
[타겟 사용자]가 [원하는 행동]을 할 것이다.
이는 [측정 지표]로 확인할 수 있다.

검증 방법: [실험/테스트 설계]
성공 기준: [구체적 수치]
```

### 3. Opportunity Solution Tree
```
목표 (Desired Outcome)
├── 기회 (Opportunity)
│   ├── 솔루션 A
│   │   └── 실험 1
│   └── 솔루션 B
│       └── 실험 2
└── 기회
    └── 솔루션 C
```

### 4. Assumption Mapping
| 가정 | 근거 수준 | 리스크 | 검증 방법 |
|------|-----------|--------|-----------|
| 사용자가 X를 원한다 | LOW | HIGH | 인터뷰 5건 |
| 기술적으로 가능하다 | HIGH | LOW | POC |
