# PM Execution Skill

프로덕트 실행 단계. 스프린트 계획, 이해관계자 관리, 릴리스 관리, 지표 추적.

## When to Activate

- 스프린트 계획/운영
- 이해관계자 보고/커뮤니케이션
- 릴리스 관리
- KPI/메트릭스 추적

## Sprint Management

### Sprint Goal Template
```
이번 스프린트에서 [사용자/시스템]이 [달성할 것]을 완료하여
[측정 가능한 결과]를 달성한다.
```

### Velocity Tracking
| Sprint | Committed | Completed | Velocity | Notes |
|--------|-----------|-----------|----------|-------|

### Burndown Analysis
- 이상적 라인 vs 실제 라인 비교
- 범위 변경(scope creep) 추적
- 병목 지점 식별

## Stakeholder Communication

```markdown
## 주간 업데이트 — [날짜]

### 요약 (3줄)
-

### 진행 상황
| 항목 | 상태 | 비고 |
|------|------|------|

### 리스크/이슈
| 이슈 | 영향 | 조치 |
|------|------|------|

### 다음 주 계획
-
```

## Release Checklist
1. 기능 완성도 확인
2. QA/테스트 완료
3. 릴리스 노트 작성
4. 이해관계자 공유
5. 모니터링 설정
6. 롤백 계획
