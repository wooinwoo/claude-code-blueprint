# 접근성 (Accessibility) 규칙

WCAG 2.1 AA 기준. 프론트엔드 컴포넌트 작성 시 항상 적용.

> 모든 접근성 규칙은 **[HIGH]**. 신규 컴포넌트는 반드시 준수, 기존 컴포넌트 수정 시 발견되면 함께 수정.

## 1. 시맨틱 HTML [HIGH]

### 올바른 태그 사용

```tsx
// BAD - div로 모든 걸 만듦
<div onClick={handleClick}>삭제</div>
<div className="heading">제목</div>
<div className="list">
  <div>항목 1</div>
</div>

// GOOD - 시맨틱 태그
<button onClick={handleClick}>삭제</button>
<h2>제목</h2>
<ul>
  <li>항목 1</li>
</ul>
```

### 랜드마크 구조

```tsx
<header>     {/* 사이트 헤더 */}
<nav>        {/* 네비게이션 */}
<main>       {/* 메인 콘텐츠 (페이지당 1개) */}
<aside>      {/* 사이드바 */}
<footer>     {/* 푸터 */}
```

## 2. 키보드 접근성 [HIGH]

### 인터랙티브 요소는 키보드 필수

```tsx
// BAD - 마우스만 가능
<div onClick={handleOpen}>열기</div>

// GOOD - 키보드 + 역할 명시
<button onClick={handleOpen}>열기</button>

// 불가피하게 div 사용 시:
<div
  role="button"
  tabIndex={0}
  onClick={handleOpen}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault()
      handleOpen()
    }
  }}
>
  열기
</div>
```

### 포커스 관리

```tsx
// 모달 열릴 때 포커스 이동
const dialogRef = useRef<HTMLDialogElement>(null)
useEffect(() => {
  if (isOpen) dialogRef.current?.focus()
}, [isOpen])

// 모달 닫힐 때 트리거 요소로 복귀
const triggerRef = useRef<HTMLButtonElement>(null)
const handleClose = () => {
  setIsOpen(false)
  triggerRef.current?.focus()
}
```

### 포커스 트랩 (모달/다이얼로그)

```tsx
// Radix UI Dialog 사용 시 자동 처리됨
<Dialog.Root>
  <Dialog.Trigger asChild>
    <button ref={triggerRef}>열기</button>
  </Dialog.Trigger>
  <Dialog.Content>
    {/* 포커스 트랩 자동 적용 */}
  </Dialog.Content>
</Dialog.Root>
```

## 3. 이미지 & 미디어 [HIGH]

### alt 텍스트 필수

```tsx
// BAD
<img src="/logo.png" />

// GOOD - 의미 있는 설명
<img src="/logo.png" alt="BidRaon 로고" />

// GOOD - 장식 이미지는 빈 alt
<img src="/decoration.svg" alt="" role="presentation" />
```

### 아이콘 버튼 레이블

```tsx
// BAD - 스크린 리더가 읽을 수 없음
<button><SearchIcon /></button>

// GOOD
<button aria-label="검색">
  <SearchIcon aria-hidden="true" />
</button>

// 또는 visually hidden 텍스트
<button>
  <SearchIcon aria-hidden="true" />
  <span className="sr-only">검색</span>
</button>
```

## 4. 폼 접근성 [HIGH]

### label 연결 필수

```tsx
// BAD
<span>이메일</span>
<input type="email" />

// GOOD - htmlFor 연결
<label htmlFor="email">이메일</label>
<input id="email" type="email" />

// GOOD - 암시적 연결
<label>
  이메일
  <input type="email" />
</label>
```

### 에러 메시지 연결

```tsx
<label htmlFor="email">이메일</label>
<input
  id="email"
  type="email"
  aria-invalid={!!errors.email}
  aria-describedby={errors.email ? "email-error" : undefined}
/>
{errors.email && (
  <p id="email-error" role="alert">
    {errors.email.message}
  </p>
)}
```

### 필수 필드 표시

```tsx
<label htmlFor="name">
  이름 <span aria-hidden="true">*</span>
</label>
<input id="name" required aria-required="true" />
```

## 5. 색상 & 대비 [MEDIUM]

### 최소 대비 비율

- **일반 텍스트**: 4.5:1 이상
- **큰 텍스트** (18px+ bold, 24px+): 3:1 이상
- **UI 컴포넌트/아이콘**: 3:1 이상

### 색상만으로 정보 전달 금지

```tsx
// BAD - 빨간색만으로 에러 표시
<span style={{ color: 'red' }}>{value}</span>

// GOOD - 색상 + 아이콘 + 텍스트
<span className="text-destructive flex items-center gap-1">
  <AlertCircle className="h-4 w-4" aria-hidden="true" />
  {value} (에러)
</span>
```

## 6. 동적 콘텐츠 [MEDIUM]

### 로딩 상태 알림

```tsx
<div aria-live="polite" aria-busy={isLoading}>
  {isLoading ? <Skeleton /> : <DataTable data={data} />}
</div>
```

### 토스트/알림

```tsx
// role="status" — 덜 긴급한 알림
<div role="status" aria-live="polite">저장되었습니다</div>

// role="alert" — 긴급한 알림 (에러 등)
<div role="alert">결제에 실패했습니다. 다시 시도해주세요.</div>
```

### 페이지 전환 알림 (SPA)

```tsx
// TanStack Router에서 라우트 변경 시
<div aria-live="polite" className="sr-only">
  {routeTitle} 페이지로 이동했습니다
</div>
```

## 7. Tailwind sr-only 유틸리티 [MEDIUM]

```tsx
// 시각적으로 숨기되 스크린 리더는 읽음
<span className="sr-only">현재 페이지:</span>

// Tailwind v4에서:
// sr-only = position:absolute; width:1px; height:1px; ...
// not-sr-only = 되돌리기
```

## 체크리스트 (코드 리뷰 시)

- [ ] 모든 `<img>`에 `alt` 있음
- [ ] 모든 인터랙티브 `<div>`에 `role` + `tabIndex` + `onKeyDown` 있음
- [ ] 모든 아이콘 버튼에 `aria-label` 있음
- [ ] 모든 폼 input에 `<label>` 연결됨
- [ ] 에러 메시지에 `aria-describedby` 연결됨
- [ ] 모달에 포커스 트랩 적용됨
- [ ] 색상만으로 정보 전달하지 않음
- [ ] 동적 콘텐츠에 `aria-live` 적용됨
